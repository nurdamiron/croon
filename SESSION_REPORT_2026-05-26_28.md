## 1. Инцидент: сайт лежал (P1001 / Server Components error)

**Симптом:** в браузере `Error in Server Components render`, в PM2-логах пачка
`PrismaClientKnownRequestError P1001: Can't reach database server`.

**Причина:** второй разработчик менял класс RDS-инстанса `alashed-db` для
оптимизации. AWS делает это с shutdown ~3 мин (видно в `aws rds describe-events`:
`DB instance shutdown` → `restarted`). Prisma-engine в работающем Next.js держит
мёртвые TCP-сокеты и **не реконнектится** сам после возврата БД.

**Лечение:** `pm2 restart alash-electronics`. Записано в memory
(`prisma-rds-reconnect.md`) — при повторе сразу рестарт, не копать сеть.

## 5. Поиск на сайте

### Фильтр по наличию
3 пилюли над результатами поиска: **Все / В наличии / Нет в наличии**. Решает
проблему «товар не в наличии теряется на 20–30 странице» — клик «Нет в наличии»
сразу показывает нужное.

### Фильтр по категории
- Селект **«искать в категории»** справа от инпута (как на ba3ar.kz) —
  аккордеон: верхние категории, под каждой раскрываются подкатегории.
  Псевдо-корень «Каталог» скрыт.
- URL `?inCat=<slug>` сужает поиск по категории + всем её потомкам (рекурсивно).
- Хлебная крошка «В категории: X ×» для сброса.
- **Баг, который ловили:** `getAllDescendantCategoryIds` возвращает только
  потомков, без самой категории → для листовой категории фильтр не срабатывал.
  Фикс: `categoryIds = [categoryId, ...потомки]`.

### История поиска
- localStorage (`alash_search_history`, до 8 запросов, без дублей).
- Дропдаун «Недавние запросы» при фокусе на пустом инпуте: иконка, текст,
  × для удаления одного, «Очистить» — всё.

## 6. Инфраструктура

### Service Worker ломал сайт при деплое
`public/sw.js` кэшировал `/_next/static/` cache-first. После деплоя имена
JS-чанков меняются → старый хеш в кэше → 404 → React hydration error #423 →
сломанный сайт (не работали интерактивные элементы).

**Фикс:** SW переписан — больше не перехватывает fetch и не кэширует. Остались
только push + notificationclick. Активная зачистка старых кэшей при первом заходе.

### Прочее
- Убран `output: 'standalone'` из `next.config.js` (несовместим с `next start`,
  ломал деплой — `Cannot find .next/standalone/server.js`).
- Добавлен `wss://mc.yandex.ru` в CSP `connect-src` (Яндекс WebVisor 2.0).

## Открытые задачи (не критичные)

- **`sharp` не установлен** в проде → Next.js image optimization не работает
  (картинки отдаются исходного размера, медленнее/дороже трафик).
- **`prisma db push` при деплое** выдаёт `Use --accept-data-loss` — в БД есть
  поле/таблица, которой нет в `schema.prisma`. Не блокирует деплой, но стоит
  разобраться (`prisma db pull` vs текущая схема).
- **Селект категории на мобилке скрыт** (`hidden md:flex`) — на телефоне поиск
  идёт по всему сайту. При желании — отдельная мобильная выпадашка.
- При смене класса RDS вторым разрабом прод сыпет P1001 ~3 мин — нужен ручной
  `pm2 restart` (или, в идеале, обернуть Prisma в reconnect-логику).
