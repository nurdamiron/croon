# Протокол кабинета Kaspi для суб-минутного демпинга

> Перехвачено вживую 2026-05-30 на боевом кабинете Alash (ID 30383258) через
> DevTools/Playwright при ручной смене цены одного товара. Это РОВНО тот механизм,
> который AlgaTop/FixBot и др. используют для суб-минутного репрайсинга.
> Домен: **mc.shop.kaspi.kz** (внутренний API кабинета продавца, НЕ публичный).

## 1. Смена цены — главный эндпоинт

```
POST https://mc.shop.kaspi.kz/pricefeed/upload/merchant/process
Content-Type: application/json
Referer: https://kaspi.kz/
x-auth-version: 3
(авторизация: куки сессии кабинета, credentials: include)

Тело:
{
  "merchantUid": "30383258",
  "availabilities": [
    { "available": "yes", "storeId": "30383258_PP1", "stockCount": 8 }
  ],
  "cityPrices": [
    { "cityId": "750000000", "value": 589 }
  ],
  "sku": "120591624_577696112",
  "model": "Модуль Wi-Fi ESP8266 ESP-01 WIFI"
}

Ответ: { "id": "6a1b2c50cfcda8501a5ebf50" }   (id операции, HTTP 200)
```

**Ключевое:** формат 1-в-1 как наш XML-фид (merchantUid, cityPrices[cityId,value],
availabilities[storeId,stockCount], sku, model) — но не файл раз в час, а **мгновенный
JSON-POST**. Применяется сразу (не ждём 60 мин перечитки фида). Вот вся суть
«суб-минутности» AlgaTop.

- `cityPrices[]` — цена ПО ГОРОДАМ (можно разные цены на город). Костанай = 750000000.
- `availabilities[]` — остатки по складам (storeId формата `{merchantUid}_PP1`).
- `sku` — merchant SKU (наш `kaspiSku` из KaspiOffer / составной PID_xxx).
- `model` — название (как kaspiName).

## 2. Цена конкурента — прямо из кабинета (БЕЗ offer-view!)

```
GET https://mc.shop.kaspi.kz/offers/api/v1/price/lowest?s=120591624
Ответ: { "price": 588.0 }
```

`s` = Kaspi PID (без суффикса). Возвращает **минимальную цену в карточке** (цена
лидера). Это альтернатива публичному offer-view — и она работает **изнутри
кабинетной сессии**, т.е. НЕ упирается в блокировку дата-центр IP (offer-view с EC2
давал 405). Если воркер логинится в кабинет, он берёт цену конкурента отсюда же.

В форме смены цены кабинет также показывает «Самая низкая цена 588 ₸» — это оно.

## 3. Сопутствующие эндпоинты (из той же сессии)

```
GET  /bff/offer-view/list?m={uid}&p=0&l=10&a=true          — список товаров продавца
GET  /bff/offer-view/details?m={uid}&s={sku}               — детали оффера
GET  /offers/api/v1/offer/count?m={uid}                    — число офферов
GET  /offers/api/v1/price/lowest?s={pid}                   — мин. цена конкурента ⭐
POST /offer-validation-api/merchant/offer/validate/v2      — валидация перед сохранением
POST /price/trends/api/v1/mc/discount                      — тренды/скидки
POST /pricefeed/upload/merchant/process                    — СМЕНА ЦЕНЫ/ОСТАТКОВ ⭐
GET  /product/stock/mc/obligated-merchant/check?m={uid}    — проверка обязательств
POST /mc/facade/graphql?opName=getOrderCounters            — счётчики заказов (GraphQL)
```

## 4. Авторизация и сессия

- Логин: `idmc.shop.kaspi.kz/login` → телефон Kaspi Pay + пароль + **SMS-код**.
- После входа — сессионные куки на `mc.shop.kaspi.kz`, заголовок `x-auth-version: 3`.
- Сессия живёт в куках; повторный SMS — при истечении/новом входе.
- **Обход SMS для бота:** Kaspi позволяет завести ОТДЕЛЬНОГО пользователя кабинета
  (раздел «Пользователи» `#/users`) с логином/паролем и правами — этому пользователю
  SMS на телефон владельца не нужен на каждый вход (так делают все боты).

## 5. Ограничения (подтверждено разведкой)

- **Лимит Kaspi: 250 операций / 30 минут на кабинет.** Каждый `pricefeed/process` =
  1 операция. Значит суб-минутный демпинг реально живёт на **~10-20 SKU**, не на всём
  каталоге. По всему каталогу — упор в лимит за минуты.
- **Серая зона ToS.** Автоматизация кабинета не запрещена явно, но Kaspi оставляет
  «ограничительные меры вплоть до прекращения сотрудничества». На боевом кабинете —
  осознанный риск владельца.

## 6. Как встроить в наш демпинг (план)

У нас уже есть движок (`lib/kaspi-dumping.ts`: computeTargetPrice) и UI. Нужен
**кабинетный executor** вместо/в дополнение к фиду:

1. **Воркер с кабинетной сессией** (на резидентной машине, как текущий
   `kaspi-dumping-worker.mjs`, но логинится в кабинет):
   - держит куки сессии mc.shop.kaspi.kz (через headless-браузер или сохранённые куки);
   - цену конкурента берёт из `/offers/api/v1/price/lowest?s={pid}` (не offer-view!);
   - считает target (наш `computeTargetPrice`);
   - применяет через `POST /pricefeed/upload/merchant/process` (тело из §1);
   - дросселирует под лимит 250/30мин (приоритет топ-SKU).
2. **Гибрид:** массовый каталог — наш XML-фид (легально, такт ~час); турбо ~10-20
   ключевых SKU — кабинетный executor (суб-минутно).
3. Креды кабинета — ТОЛЬКО в локальном `.env` воркера на машине владельца, не в репо.

## 7. Открытые вопросы для проверки

- Точное время жизни сессии (когда снова просит SMS).
- Работает ли отдельный «пользователь-робот» без SMS на каждый вход (создать в #/users).
- Реальная скорость до применения на витрине (форма обещает «до 15 мин», но pricefeed
  может быть быстрее — проверить замером).
- Реакция Kaspi на серию быстрых pricefeed-запросов (детект/троттл до 250).
